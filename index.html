Infix notation to PharmML math online converter

Toni Giorgino at isib.cnr.it

This web service converts a mathematical expression in the usual infix notation into the corresponding PharmML math-XML markup. For details and source code see the infix2pharmml project page at SourceForge.

Version 0.5

What's new

You can the following features are now supported:
  • Constants (and user-friendly aliases, such as e for exponentiale)
  • Variable assignments, such as a=b+c (assignment) and a:=b+c (definition)
  • More user-friendly function call syntax
  • All operators defined by PharmML 0.3
  • Factorial operator (!)

Limitations

For a full list of the converter's limitations, see the project's README. As a non-exhaustive summary, note the following ones:
  • Use ln(...) for the natural logarithm function (requires PharmML ≥ 0.3).
  • Logical operators are not supported.
  • User-defined function calls (with named arguments) are coded like: combinedErrorModel(a=a,b=b,f=Cc)
