Infix notation to PharmML math online converter

Toni Giorgino at isib.cnr.it

This web service converts a mathematical expression in the usual infix notation into the corresponding PharmML math-XML markup. For details and source code see the infix2pharmml project page at SourceForge.

Limitations

For a full list of the converter's limitations, see the project's README. As a non-exhaustive summary, note the following ones:
  • The expression syntax is defined by the Math::Symbolic parser. Some of its functions do not match those defined by PharmML.
  • In particular, the log function is always two-argument.
  • Some conversions are performed automatically, e.g. sqrt -> ^ 0.5
  • Logical operators are not supported.
  • User-defined function calls (with named arguments) are coded like: call(combinedErrorModel,a=a,b=b,f=Cc)

Enter the formula below:

Examples:

1+sqrt(a^2+b^2)

call(myFunction,arg1=sin(x),arg2=2)

log(x,2.7)

A*exp(-alpha*t)+B*exp(-beta*t)
